# Segurança

Se você encontrar uma falha de segurança no ZashCRT, não abra uma issue pública com detalhes sensíveis.

Entre em contato com Leonardo Berbert por um canal privado antes de publicar informações que possam expor usuários, credenciais ou ambientes corporativos.

## Ao reportar

Inclua, quando possível:

- versão afetada
- sistema operacional
- impacto esperado
- passos de reprodução sem credenciais reais
- arquivos ou logs anonimizados

## Nunca envie publicamente

- senhas
- chaves privadas
- tokens
- hosts internos
- scripts com credenciais
- prints contendo informações corporativas sensíveis
